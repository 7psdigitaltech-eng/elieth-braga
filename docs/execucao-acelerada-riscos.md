# Riscos da execução acelerada

## Riscos principais

- mudanças grandes demais para revisão;
- arquivos fora do escopo;
- conteúdo factual não validado;
- aparência eleitoral involuntária;
- dependências adicionadas sem necessidade;
- perda de rastreabilidade;
- mistura entre implementação, deploy e infraestrutura.

## Mitigações

- prompt fechado e explícito;
- lista de arquivos permitidos e proibidos;
- proibição de commit automático;
- relatório final obrigatório;
- screenshots obrigatórios para interface;
- revisão por diff antes do commit;
- PR pelo GitHub antes de merge;
- divisão da entrega se o diff ficar amplo demais.
