# Critério de parada

Durante execução acelerada, o Codex deve parar se:

- precisar alterar arquivo proibido;
- precisar instalar dependência;
- encontrar conflito entre documentos;
- precisar decidir conteúdo factual;
- precisar decidir design além do escopo;
- encontrar risco eleitoral, autoral ou de privacidade;
- comando obrigatório falhar.

Parar significa reportar o problema e aguardar validação.
